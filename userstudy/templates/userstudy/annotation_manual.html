Table Annotation Instruction manual

Here we provide a step by step guide on how to annotate quantity mentions!

Step one

Open the URL: http://credi.mpi-inf.mpg.de:8000/userstudy/next/, and login to the application using the credentials provided by your supervisor.
If you've completed all the documents assigned to you, you will see the following page:
Then you should contact your supervisor to assign more documents to you.

Step two

After logging into the tool, you will see the annotation page as follows:
As you see it displays a document with textual content and a single related table. The text has a highlighted quantity mention that we need to know its relation to the given table. The text will be displayed in the top of the page and the table will be displayed in the right under the text. While the annotation options are displayed in the left side of the page.

Step Three

You should select the correct annotation for the highlighted quantity mention from the given options. Then submit your answer using the 'Submit button'.
Make sure you select the correct annotation option as by clicking on submit you won't be able to change your answer.
After submitting your answer a new document that need to be annotated will be displayed.

Explaining The Annotation Options

Please make sure you fully understand the meaning of each option, when in doubt please contact Yusra Ibrahim .

Not a Mention

If the mention is not a 'true' quantity, for example the '2' in CO2 is not a quantity mention neither is the 6 in Apple iPhone 6. We define a quantity mention as any number that quantifies a certain aspect of the world. A quantity mention may have an associated units such as money '$25', percentage '10%' and speed '10 km/hr'; or may represent a simple count such as '400 people'. We don't consider the following as quantity mentions:
  1. Date and Time
  2. Headings, for example: Section 2, Table 1.3, Fig. 6, etc.
  3. Phone Numbers
  4. Reference Numbers, for example: in [2], §25A(a)
Thus if you encounter any of these you should mark as Not a Mention.

Not Related

If the quantity in text represents a correct quantity mention, however it is not related to the given table, then you should select Not related.

If the highlighted quantity mention is related to the given table, please select the type of relation from the given options and mark all the table cells that are associated with this quantity. The following is the type of interesting relations:

General Notes

  • Read the text carefully and understand its relation to the table before annotating the quantity mention.
  • Make sure not to confuse difference with decrease/increase ratio.
  • Make sure to select the correct relation and all its related cells in the table.
  • If the highlighted quantity mention in text is an approximation of a quantity in table, select "Approximate" along with the correct relation, i.e. Single Value, Sum, Difference, etc. While if the highlighted quantity mention is an exact value of a quantity in table, select "Exact" along with the correct relation.
  • Check the correctness of your answer before submitting.
  • Pay attention to the units, the decimal points and number scale. For example, Monetary values that are given in thousands USD$ in table and in million USD$ in text. Also, two quantities with the same value and different units are not related. For example, 2.4% is not related to 2.4$.

FAQ

Can I use the browser back button to edit a submitted answer?

No, please submit your answer once and don't use the browser back button.

Can a quantity-table pair be related their context does not match? For example, when the table present data from different year than that given in the text?

No, the context must match!

What if the context matches but the numbers do not match?

Then it is unrelated. We only consider a quantity-table pair related if the context and the numbers match.

What is the target of this study?

This study is mainly about quantities, thus the selected cells+relation must match the highlighted number in text.

How do you define an approximate match?

You should use your common sense. Approximations should be reasonable in terms of rounding up or down; for example 70,000 is an approximation of 70,100 or 69,890, but it is not approximation of 69,000 nor 68,900. However, sometimes you will find a quantity mention of the form "almost 25,000" matching quantity "24,449" in table. Thus, it depends on the context and the words used!

If the relation between the quantity mention in the text and the table is not obvious, Should I find a complex mathematical relation between them?

No, don't try to come up with clever formula if the numbers do not match from the first glance.

If I have two possible relations between the quantity mention in text and the table, which one should I chose?

You should chose the simplest most obvious relation, that is the explicitly mentioned quantity. Then, in the comment you can write the other (implicit) relation as follows:

  • When the quantity mention is an aggregation over the cells in the column, then write in the comment: "col-AggregateFunctionName".
  • When the quantity mention is an aggregation over the cells in the row, then write in the comment: "row-AggregateFunctionName"
  • In case the aggregation is only performed on a subset of the cells in this row/column, then add "[start:end]", where start and end are the one-based indexes marking the range of cells to which the aggregation applies. For example "row-sum[3-6]"
  • You can use the AggregateFunctionName as follows: sum,diff,rat,avg,max,min. Or use another function name if not given in this list

Why do I see the same quantity multiple times?

Please note that a quantity in the text might be paired with multiple tables from the same page, in such cases if you choose "Not Related" the next paired table will be displayed.

When should I use the skip button?

In very rare cases at which the text and/or table does not make sense. For example if the page has non-english content. Skipping a mention/document will skip it for all the annotators

What about quantity mentions with wrong boundaries? for example " 6 500 worker" or "increased from 456 to 618"

In this case annotate the correct mention in the first case "6500" and in the second case "456", then write in the comment "WB" which means wrong boundary

What if the quantity mention in the text is giving a range of values, for example "in the range of 56 to 90?

Then, select the related cells at which this range applies and select "other" relation with value "range"

What if the quantity mention is part of a row/column header, such as " 1-14 years"? what if it has a wrong boundaries, such as " 16 to 24 hours"?

Annotate it as "other" with value "header", and if it has a wrong boundary add "WB" to the comment.You can alos mark it as "Not a Mention"!
